## Setup

### Standard Vue 3 / Vite Project

The plugin automatically injects DevTools setup in your application:

```ts [vite.config.ts]
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { VueAirportDevTools } from 'vue-airport-devtools/vite'

export default defineConfig({
  plugins: [
    vue(),
    VueAirportDevTools(), // Auto-injects DevTools
  ],
})
```

::tip
Auto-injection only adds DevTools in development mode (`DEV`).
::

### Nuxt Project

For Nuxt, simply add the DevTools module to your configuration:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: [
    // ... other modules
    'vue-airport-devtools/nuxt',
  ],
})
```

::warning
Development tools do not integrate in Nuxt devtools panels. For the time being, use the standard Vue DevTools extension in the browser developer tools.
::

### Manual Setup (without Vite)

If you prefer manual setup or don't use Vite:

```ts [main.ts]
import { createApp } from 'vue'
import { setupAirportDevTools } from 'vue-airport-devtools'
import App from './App.vue'

const app = createApp(App)

// Enable DevTools only in development
if (import.meta.env.DEV) {
  setupAirportDevTools(app)
}

app.mount('#app')
```

## Usage

### Enabling DevTools for a Desk

DevTools integration must be explicitly enabled when creating a desk:

```ts
import { useCheckIn } from 'vue-airport'

const { createDesk } = useCheckIn()

createDesk(MY_DESK_KEY, {
  devTools: true, // Enable DevTools for this desk
})
```

::warning
DevTools are **opt-in** per desk. Set `devTools: true` in the desk options to enable tracking and visualization.
::

### Identifying Your Desks

For better visibility in DevTools, you can optionally add a descriptive `deskId`:

```ts
createDesk(MY_DESK_KEY, {
  devTools: true,
  deskId: 'my-custom-desk', // Optional: better visibility in DevTools
})
```

::tip{icon="i-lucide-lightbulb"}
The `deskId` is optional. Without it, an auto-generated identifier will be used in DevTools.
::

### Adding Metadata

Item metadata appears in the inspector for easier debugging:

```ts
const { checkIn } = useCheckIn(MY_DESK_KEY)

checkIn('item-1', myData, {
  label: 'My Main Item',
  active: true,
  description: 'Description for DevTools',
})
```

## Registry Inspector

The **Airport Registry Inspector** lets you visualize the complete structure of your registries.

### Tree View

- **Desks**: Displayed with their `deskId` or custom label
- **Badges**: Indicate the number of registered children
- **Items**: Listed under each desk with their identifiers
- **Tags**: Active items are marked with an "active" badge

### State Details

Select a node to see:

- **Metadata**: All item metadata (label, active, custom fields)
- **Data**: Current item data
- **Context**: Parent desk context
- **Timestamps**: Creation and last update dates
- **Registry Info**: Registry statistics (item count, etc.)

:::code-group
```ts [State Example]
{
  metadata: {
    label: "Todo Item #1",
    active: false,
    priority: "high"
  },
  data: {
    title: "Buy milk",
    completed: false
  },
  timestamps: {
    createdAt: "2025-01-15T10:30:00.000Z",
    updatedAt: "2025-01-15T11:45:00.000Z"
  }
}
```
:::

### Event Details

Each event displays:

- **Desk ID**: Identifier of the concerned desk
- **Child ID**: Item identifier (if applicable)
- **Plugin Name**: Name of the executed plugin (for plugin events)
- **Duration**: Plugin execution time in milliseconds
- **Timestamp**: Exact time of the event
- **Data**: Data before/after modification

::tip{icon="i-lucide-filter"}
Use timeline filters to display only specific event types.
::

## Debugging with DevTools

### Scenario 1: Verify Registration

If a child component doesn't seem to communicate with its parent:

1. Open the **Registry Inspector**
2. Check that the desk exists in the tree
3. Check that the item appears under the desk
4. Inspect the item's metadata and data

### Scenario 2: Trace Updates

To understand when and how an item is updated:

1. Open the **Timeline**
2. Interact with your component
3. Observe the `â†» Update` events in the timeline
4. Click on an event to see before/after data

### Scenario 3: Debug Plugins

To verify plugin execution:

1. Observe `âš¡ Plugin` events in the timeline
2. Check the **execution duration** (performance)
3. Inspect data transformed by the plugin
4. Verify the execution order of multiple plugins

:::code-group
```ts [Plugin Example]

const debugPlugin = {
  name: 'debug',
  execute: (context, data, meta) => {
    console.log('[Plugin Debug]', { data, meta })
    return data
  }
}

createDesk(MY_DESK_KEY, {
  devTools: true,
  deskId: 'debug-desk',
  plugins: [debugPlugin],
})

// Will appear in timeline with:
// âš¡ Plugin: debug
// Duration: 0.5ms
```
:::

## Performance

DevTools are automatically disabled in production and have **no impact** on your final bundle.

::warning{icon="i-lucide-zap"}
Timeline events are kept in memory. For apps with heavy activity, consider refreshing DevTools periodically.
::

## Compatibility

- **Vue DevTools 6+**: Recommended version
- **Vue 3.3+**: Required for VueAirport
- **Browsers**: Chrome, Firefox, Edge (with Vue DevTools extension)

## Troubleshooting

### DevTools Don't Appear

1. Check that Vue DevTools is installed in your browser
2. Verify you're in development mode (`npm run dev`)
3. Check the configuration in your `vite.config.ts` or Nuxt plugin
4. Open the console to see any errors

### Events Are Not Tracked

1. Check that the desk has a `deskId` defined
2. Ensure DevTools are initialized before desks
3. Check for errors in the console

### Inspector Is Empty

1. Create at least one desk with `createDesk()`
2. Register at least one item with `checkIn()`
3. Refresh the inspector with the refresh button
